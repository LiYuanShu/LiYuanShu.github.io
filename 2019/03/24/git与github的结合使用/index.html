git与github的结合使用

Posted on 2019-03-24

git为版本控制工具，github为代码托管平台。可以使用git把自己的一些项目上传到github上，或者创建一个项目发布到github，进行协同开发。

git命令

创建一个文件夹，并且在git进入到文件中，初始化成为本地创库

把所需要上传的项目复制到工作区中，然后添加到缓存区，提交到本地仓库


查看git连接github远程创库所使用的别名，若没有则可以增加，方便上传

推送到github远程创库,由网速决定，如果项目大，可能需要一定时间

克隆远程库

首先新建一个文件夹
执行命令：git clone URL (URL为远程仓库地址)
执行命令后会有3个效果：
完整把远程库下载到本地
创建origin远程地址别名
初始化本地库

协同管理仓库

用自己的账号登录到github，进入到需要协同开发的仓库，然后点击Settings——>Collaborators，输入邀请成员的github账号，就可以得到一个邀请链接，然后把邀请链接发给别邀请成员就可以了

被邀请的成员需要打开链接，点击同意加入开发团队，完成。

pull=fetch+merge

git fetch origin master：抓取远程仓库内容，但这时候本地文件内容不会直接更新。
git checkout origin/master：切换分支，查看抓取的内容
git merge origin/master:合并分支，把新内容合并到本地仓库

Django项目文件作用

manage.py:和项目交互都基于这个文件。可以在终端输入python manage.py [命令]。用python manage.py help查看可以进行哪些操作
settings.py: 项目的设置项，和项目相关的配都在里面
urls.py: 配置URL路由，url与视图函数做映射。例如访问http://127.0.0.1/news/ 是访问新闻列表页，这些东西需要在这个文件完成
wsgi.py: 项目与WSGI协议兼容的web服务器入口，部署的时候需要用到的，一般情况不用修改

Django推荐项目规范

安装功能或者模块分层，分成一个个APP，把每个模块的视图写在对应的APP的view.py中，方便管理
cmd：python manage.py [APP名称]

DEBUG模式

在setting中设置debug模式，true为开启，false为关闭。
①开启后在项目发生修改时，ctrl+s就可重新启动django项目
② django项目代码出现bug，在浏览器和控制台中会出现错误信息
③项目上线后，需关闭debug模式，否则有安全隐患

APP的添加

在django网站项目中，可以分为很多个模块，每个模块对应一个APP，这样可以更方便管理
在cmd创建一个APP

创建APP后，在views.py创建需要的内容。同时也要在urls.py的添加路径

git命令

创建一个文件夹，并且在git进入到文件中，初始化成为本地创库

把所需要上传的项目复制到工作区中，然后添加到缓存区，提交到本地仓库


查看git连接github远程创库所使用的别名，若没有则可以增加，方便上传

DNS服务的搭建与配置

Posted on 2019-03-10

DNS服务器主要提供域名解析功能，例如把www.baidu.com这个域名解析为相对应的IP地址，然后才能对百度进行访问。
两种解析方式：
递归查询：客户机向根服务器进行查询，后面的查询操作由各级DNS服务器相互进行，最后由根服务返回结果给客户机
特点：客户机只需要发出一次请求，便可以得到域名对应的IP地址
迭代查询：客户机向根服务器进行查询，若根服务器无法解析，便返回一级域名服务器给客户机，然后客户机再向一级域名服务器查询，以此类推，直到得到IP地址
特点：客户机可能需要发出多次请求，和多个DNS服务器进行通信

安装

需要安装包：bind-9.9.4-50.el7.x86_64.rpm
本次安装首先采用的是rpm包安装方式。出现问题：此rpm包有较多依赖包，无法已持续安装。解决方案：使用yum安装。

#####把yum源设为光盘驱动的rpm包
进入此目录会发现很多配置文件，可以把这些文件copy到别的文件夹，已备不时之需
[root@test ~]# cd /etc/yum.repos.d/
新建一个配置文件，并进行一些信息配置：
[root@test yum.repos.d]# vim dvd.repo

清除缓存：[root@test yum.repos.d]# yum clean all
查看yum源：[root@test yum.repos.d]# yum list |grep bind
安装：[root@test yum.repos.d]# yum install bind.x86_64
启动dns服务：systemctl start named.service
查看服务是否运行：netstat -nul ————————》监听端口为53

①named.conf ：主配置文件（定义DNS服务的运行）
②named.rfc1912.zones：区域配置文件（保存IP与域名对应关系的文件所在位置）
③ 由区域配置文件指定位置,一般在/var/named目录下:数据配置文件（详细记录IP与域名的对应关系）

git版本切换

Posted on 2019-03-05

git本地库经过修改本地内容后，可以无数次提交到本地仓库，通过秘钥记录的方式，可以自由切换到自己想要的版本。

查看日志

git reflog：查看版本号，可显示所有版本记录
git log –oneline：只显示当前版本的历史记录

基于索引值：

①git reflog：查看版本号（第一列）

②git reset –hard 版本号

使用^符号：

git reset –hard HEAD^ (HEAD为指针，指向当前使用版本) 

一个^符号只能回退到前一个版本，可以使用多个^,只能回退，不能前进

使用~符号：

git reset --hard~n
    可一次性回退n个版本

参数对比

git reset –hard：在本地库移动指针，重置暂存区，重置工作区
–mixed：在本地库移动指针，重置暂存区
–soft：仅在本地库移动指针

文件比较

git diff filename:比较工作区的文件(在第一行添加一个2)

git diff HEAD filename (与历史版本比较 HEAD为某个历史版本)

分支管理

分支：在版本控制过程中，使用多条线同时推进多个任务，加快开发速度。
实验包括两个分支：主支master，分支hot_fix
git branch -v:查看分支
git branch branchName:创建分支
git checkout branchName：切换分支

合并分支

①切换到接受修改的分支上（git checkout master）
②进行合并操作（git merge hot_fix）


注：内容冲突：两个分支修改同一个地方，内容不同，需要人为沟通，进行文件内容修改

删除内容

①git rm fileName
②git add *
③git commit

dhcp服务的搭建和配置

Posted on 2019-03-03

简介

DHCP：动态主机配置协议（dynamic host configuration protocol）
dhcp服务作用：管理动态的ip地址分配以及其他相关的环境配置工作
实现过程：①客户端————dhcp discover————>服务端
②客户端<————dhcp offer————服务端
③客户端————dhcp request————>服务端

④客户端<————dhcp acknowledge————服务端

任务

内部网段为192.168.1.0/24,路由器ip为192.168.1.1
dhcp分配的ip地址默认租期为一天，最长为两天
ip地址范围为192.168.0.100~192.168.0.250
为某台特定的mac指定特定ip地址

安装

查看linux系统是否安装dhcp服务：[root@test ~]# rpm -qa |grep dhcp
如果只有这两个包则表明好没有安装dhcp服务，需要安装
dhcp-libs-4.2.5-58.el7.centos.x86_64——————》库
dhcp-common-4.2.5-58.el7.centos.x86_64——————》命令

①把linux安装盘挂载到某个空目录下
[root@test ~]# mount /dev/cdrom /mnt/cdrom/ **确保存在/mnt/cdrom目录

②查看dhcp服务包的具体名称
[root@test Packages]# cd /mnt/cdrom/Packages/
[root@test Packages]# ll | grep dhcp
-rw-rw-r–. 2 root root 525484 8月 10 2017 dhcp-4.2.5-58.el7.centos.x86_64.rpm
-rw-rw-r–. 3 root root 177816 8月 10 2017 dhcp-common-4.2.5-58.el7.centos.x86_64.rpm
-rw-rw-r–. 3 root root 133148 8月 10 2017 dhcp-libs-4.2.5-58.el7.centos.x86_64.rpm

③安装dhcp服务
[root@test ~]# cd /mnt/cdrom/Packages/
[root@test Packages]# rpm -ivh dhcp-4.2.5-58.el7.centos.x86_64.rpm

配置

两个重要文件
主配置文件(/etc/dhcpd.conf)：一套声明集和一套参数集
ip地址分配信息(/var/lib/dhcpd/dhcpd.leases)：记录已经分配的ip地址和客户端信息

配置文件信息如下：vim /etc/dhcp/dhcpd.conf

客户端

让客户端使用上自己配置的dhcp服务，需要进行一些配置
①配置mvware软件，不要勾选，其中nat模式和主机模式都可以，桥接模式不行

②把服务器和客户端都设置为同一种模式下（主机模式）

客户端的网卡配置文件：vi /etc/sysconfig/network-scripts/ifcfg-ens33

重新启动网卡：service network restart
查看日志信息：cat cat /var/lib/dhclient/dhclient.leases

疑点：当在dhcp服务端设置为MAC地址分配特定地址时，在客户端物理重启多少次网卡，也无日志记录！！
